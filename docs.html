Image Optimization API

Comprehensive image processing with compression, conversion, resizing, and watermarking

About

A powerful image processing API built with Node.js and Sharp that provides comprehensive image optimization features including compression, format conversion, resizing, and watermarking capabilities.

Tech Stack

Node.js Express.js Sharp Multer Axios MIME

API Endpoints

Compression

POST /compress
Compress image with quality control
image (file) - Image file to compress
quality (number) - Quality 1-100, default: 100
POST /compress/url
Compress image from URL
imageUrl (string) - URL of image to compress
quality (number) - Quality 1-100, default: 100

Format Conversion

POST /convert/webp
Convert image to WebP format
image (file) - Image file to convert
quality (number) - Quality 1-100, default: 80
POST /convert/webp/url
Convert image to WebP from URL
POST /convert/avif
Convert image to AVIF format
POST /convert/avif/url
Convert image to AVIF from URL

Resizing

POST /resize
Resize image with custom dimensions
image (file) - Image file to resize
width (number) - Target width (required)
height (number) - Target height (optional)
fit (string) - Fit mode: cover|contain|fill|inside|outside
POST /resize/url
Resize image from URL
POST /resize/thumbnail
Generate square thumbnail
image (file) - Image file
size (number) - Thumbnail size, default: 150
POST /resize/thumbnail/url
Generate thumbnail from URL

Watermarking

POST /watermark/text
Add text watermark
image (file) - Image file
text (string) - Watermark text (required)
fontSize (number) - Font size, default: 24
color (string) - Text color, default: white
opacity (number) - Opacity 0-1, default: 0.7
position (string) - Position: top-left|top-right|bottom-left|bottom-right|center
margin (number) - Margin from edge, default: 20
POST /watermark/text/url
Add text watermark from URL
POST /watermark/image
Add image watermark
image (file) - Main image file
watermark (file) - Watermark image file
opacity (number) - Opacity 0-1, default: 0.7
position (string) - Position, default: bottom-right
margin (number) - Margin from edge, default: 20
scale (number) - Watermark scale 0-1, default: 0.2
POST /watermark/image/url
Add image watermark from URL

Legacy Endpoints

POST /optimize
Basic image optimization (legacy)
POST /optimize-url
Optimize image from URL (legacy)

Response Format

All endpoints return a consistent JSON response:

{
  "url": "/optimized/filename-action.extension",
  "fullUrl": "http://localhost:8080/optimized/filename-action.extension"
}

File Naming Convention

Processed images follow the pattern: filename-{action}.{extension}

filename-compressed.jpg
filename-webp.webp
filename-resized.jpg
filename-thumbnail.jpg
filename-watermark.jpg

Supported Formats

Input: JPEG, PNG, WebP, GIF
Output: JPEG, PNG, WebP, AVIF (depending on endpoint)

Example Usage

# Compress image
curl -X POST http://localhost:8080/compress \
  -F "image=@photo.jpg" \
  -F "quality=80"

# Convert to WebP from URL
curl -X POST http://localhost:8080/convert/webp/url \
  -H "Content-Type: application/json" \
  -d '{"imageUrl": "https://example.com/image.jpg", "quality": 80}'

# Resize image
curl -X POST http://localhost:8080/resize \
  -F "image=@photo.jpg" \
  -F "width=800" \
  -F "height=600"

# Add text watermark
curl -X POST http://localhost:8080/watermark/text \
  -F "image=@photo.jpg" \
  -F "text=Copyright 2024" \
  -F "position=bottom-right"
